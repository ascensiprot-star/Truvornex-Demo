### Typography
- **Primary font:** Inter (Google Fonts)
- **Monospace:** JetBrains Mono (for data, codes, IDs)
- **Type scale:** Display 32px / H1 24px / H2 20px / H3 16px / Body 14px / Caption 12px / Label 11px
- **Letter spacing:** Headings use `0.02em`, labels use `0.08em` (uppercase tracking)
- **Weight:** 300 (light body), 400 (regular), 500 (medium), 600 (semibold), 700 (bold)

### Motion
- Spring easing for interactive elements: `Curves.elasticOut` equivalent
- Expo out for transitions: fast entry, slow settle
- Page transitions: slide from right (standard), fade+scale for modals
- Micro-animations on tap: slight scale down (0.97) then back

### Elevation / Shadows
Cards and surfaces use layered box shadows with deep blacks — not Material elevation grays. Borders are `rgba(255,255,255,0.07)` hairlines.

## SUPABASE INTEGRATION

```dart
// supabase_client.dart
await Supabase.initialize(
  url: const String.fromEnvironment('SUPABASE_URL'),
  anonKey: const String.fromEnvironment('SUPABASE_ANON_KEY'),
);

// All entity repos follow this pattern:
class BookingRepository {
  final SupabaseClient _client;
  
  Future<List<Booking>> getMyBookings(String userId) async {
    final data = await _client
      .from('bookings')
      .select('*, provider:providers(*), service:services(*)')
      .eq('customer_id', userId)
      .order('created_at', ascending: false);
    return data.map(Booking.fromJson).toList();
  }
  
  Stream<List<Booking>> watchMyBookings(String userId) {
    return _client
      .from('bookings')
      .stream(primaryKey: ['id'])
      .eq('customer_id', userId)
      .map((data) => data.map(Booking.fromJson).toList());
  }
}
```

Realtime subscriptions for: Chat, TrackService, NotificationCenter, EmergencyRequest.

## WHAT "LAB-GRADE" MEANS IN FLUTTER

Every detail must signal precision engineering:

1. **No default Material widgets visible** — override everything. No blue ripples, no default AppBar shadows, no standard SnackBar styling. Every widget is custom-skinned.

2. **Haptic feedback** everywhere meaningful — `HapticFeedback.lightImpact()` on taps, `HapticFeedback.mediumImpact()` on confirmations, `HapticFeedback.heavyImpact()` on destructive actions.

3. **Skeleton loaders not spinners** — every list, card, and stat uses `Shimmer` while loading. The spinner only appears on full-screen blocking operations.

4. **Pull-to-refresh** on every scrollable list — with custom refresh indicator styled to match the monochrome theme.

5. **Empty states are designed** — not "No data found". Each empty state has a minimal icon, a headline, and a CTA. Example: empty bookings → "Nothing booked yet" → "Browse services" button.

6. **Error states are designed** — inline, not crashes. Retry button. Network error shows an offline indicator in the app bar.

7. **Transitions are intentional** — hero animations on provider cards → provider detail. Shared element transitions on service images.

8. **Typography is tight** — no default Flutter font sizes. Every `Text` widget uses a named style from `AppTypography`. Zero inline `fontSize` scattered through widgets.

9. **Bottom sheets over new screens** — search, filters, quick actions, confirmations all use `showModalBottomSheet` with custom drag handle, rounded top corners (`BorderRadius.vertical(top: Radius.circular(20))`), dark background.

10. **Safe area and notch handling** — all screens use `SafeArea`. Bottom nav accounts for home indicator on iPhone.
