## 2. Supabase Edge Functions (Deno, in `supabase/functions/`)

1. **`simon-dispatch`** — replaces direct DeepSeek calls. Takes a prompt + context (zone demand/supply data), holds the LLM key as a Supabase secret, returns structured JSON (recommendations, matches, dispute summaries). Rate-limit per user_id.
2. **`emergency-match`** — given an `emergency_requests` row, queries `zone_members` + `ProviderMetrics` for currently-idle providers in radius (PostGIS `ST_DWithin`), ranks by distance + trust score, triggers Notification + Realtime broadcast.
3. **`group-buy-lock`** — triggered when `group_buy.current_participants >= target_participants`; assigns best-fit provider, updates status to `locked`, creates `Booking` rows for each participant.
4. **`predictive-maintenance-scan`** — scheduled (pg_cron daily), scans booking history per customer per category, writes `predictive_maintenance_signals`.
5. **`receipt-story-generate`** — on booking completion, calls `simon-dispatch` to generate the shareable summary text, writes `receipt_stories`.
6. **`jury-resolve`** — when a dispute reaches quorum (3 votes), tallies votes, updates `disputes.status`, distributes `time_credits_ledger` rewards to jurors.
